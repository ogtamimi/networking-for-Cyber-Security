# Contributing Guide

We believe in community-driven learning and continuous improvement. If you spot a typo, want to improve an explanation, or have a great idea to enhance any chapter - your contribution is welcome.

## How to Contribute

1. Fork the repository  
2. Clone your fork  
```bash
git clone https://github.com/your-username/networking-for-Cyber-Security.git
````

3. Create a new branch

```bash
git checkout -b feature/awesome-addition
```

4. Make your changes

* Edit the relevant `.md` file inside the `Chapters/` folder
* Keep explanations clear, structured, and beginner-friendly

5. Commit your changes

```bash
git commit -m "Improve Chapter X: add explanation about VLAN Hopping"
```

6. Push your branch

```bash
git push origin feature/awesome-addition
```

7. Open a Pull Request

* Describe what you changed and why

## Contribution Rules

* Keep content accurate and easy to understand
* Avoid unnecessary complexity or copy-pasted material
* Maintain the existing chapter structure and style
* Focus on networking concepts relevant to cybersecurity

## Thank You

Thank you for contributing to **Networking for Cyber Security**. Every improvement helps build a stronger learning resource for everyone.
